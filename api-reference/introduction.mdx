# Wool Ball API Reference

<Note>
Welcome to the Wool Ball API reference. Our API provides access to a distributed network of browser-based AI processing capabilities.
</Note>

## Base URL

```bash
https://api.woolball.xyz/v1
```

## Authentication

<CodeGroup>
```bash
curl -X POST https://api.woolball.xyz/v1/text-to-speech \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello World"}'
```

```javascript
fetch('https://api.woolball.xyz/v1/text-to-speech', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    text: 'Hello World'
  })
})
```

```python
import requests

response = requests.post(
    'https://api.woolball.xyz/v1/text-to-speech',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json',
    },
    json={
        'text': 'Hello World'
    }
)
```
</CodeGroup>

## Rate Limits

<Accordion title="Free Tier Limits">
- 100 requests per hour
- Maximum text length: 1000 characters per request
- Maximum audio length: 1 minute per request
- Maximum image size: 5MB per request
</Accordion>

<Accordion title="Paid Tier Limits">
- Based on your subscription level
- Custom limits available for enterprise plans
- Network contributors receive additional quota
</Accordion>

## Available Endpoints

### Text Processing
- `/text-to-speech`: Generate natural speech from text
- `/speech-to-text`: Convert audio to text
- `/text-generation`: Generate AI-powered text content
- `/translation`: Translate text between languages
- `/zero-shot-classification`: Classify text without predefined categories
- `/summarization`: Create concise text summaries

### Image Processing
- `/facial-emotion`: Analyze emotions in facial images
- `/image-text-to-image`: Generate images from text and image input
- `/image-classification`: Classify image content
- `/zero-shot-image-classification`: Flexible image classification
- `/character-to-image`: Generate images from characters/emojis

## Response Format

<CodeGroup>
```json Success Response
{
  "success": true,
  "data": {
    // Response data specific to the endpoint
  },
  "error": null
}
```

```json Error Response
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message"
  }
}
```
</CodeGroup>
